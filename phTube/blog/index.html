CONST

Constants are block level scope variables. The scope may either global or local scope. Constants are working like variables defined using let statement.We can't modify value of constants. Constants re-declarations are not allowed.

Syntax

const PI = 3.14159;
PI = 3; // This will result in an error because you can't reassign a constant variable
const person = { name: 'John' };
person.name = 'Jane';
// This is allowed, as we are modifying the property, not reassigning the whole object

Javascript NULL and UNDEFINED

Null

Null is a value that represents the intentional absence of any object value. It's often used to indicate that a variable should have no value or that a value doesn't exist yet.

Use null when you want to explicitly set a variable to have no value.

Null is used to indicate that something is intentionally empty or non-existent.

Example: let variableName = null;

Undefined

Undefined is a value that represents the absence of a value in a variable. It typically indicates that a variable has been declared but hasn't been assigned a value.


Undefined is often automatically assigned to variables that have been declared but not initialized, or when trying to access a non-existent property of an object.

Undefined is often the result of variables that haven't been initialized or accessed improperly.

What is a REST API?

REST API

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing web services. REST APIs use HTTP methods to perform operations on resources.

Representational State Transfer (REST) is an architectural style for designing distributed hypermedia systems. It defines a set of constraints that systems must adhere to in order to be considered RESTful.
Application Programming Interface (API) is a set of rules that define how two software applications can interact with each other.
HTTP methods are the verbs used in HTTP requests. The most common HTTP methods for REST APIs are GET, POST, PUT, and DELETE.
